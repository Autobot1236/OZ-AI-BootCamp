# GitHub & HTML
- GitHub

    - Git 저장소를 호스팅하는 서비스
    - 계정을 만든 뒤 새 Repository를 생성하면 원격 저장소 주소(URL)가 발급

    >git remote add origin "원격주소"
    
    >git remote -v
    
    - git push: 로컬 커밋을 원격 저장소로 올림
    - git pull: 원격 변경 사항을 내려받아 로컬에 반영
    
        1. 로컬 커밋을 원격 저장소로 업로드
            >git push -u origin main
        2. GitHub 웹에서 파일 추가 & 커밋 생성
        3. 원격 저장소의 변경 사항 다운로드
            >git pull
- Clone
    - 원격 저장소를 그대로 내 로컬로 복사
    - 같은 저장소에 push 권한이 있을 때 사용
    - 팀 내부 협업에 주로 사용
- Fork
    - 다른 사람의 저장소를 내 GitHub 계정으로 복사
    - 원본 저장소에 직접 push 권한이 없을 때 사용
    - 오픈소스 기여 방식
- PR 
    
    원본에 내 변경사항 제안
- Pull Request
    
    변경 사항을 원본 저장소에 반영해달라고 요청하는 것
    - 변경 내용 설명
    - 리뷰 요청
    - 병합 전 검증 단계
- Code Review
    
    코드를 병합하기 전에 함께 확인하는 과정
    - 버그 예방
    - 코드 품질 유지
    - 팀 규칙 공유

    <기본 원칙>
    - 코드 중심으로 리뷰
    - 이유를 남긴 코멘트
    - 승인 후 병합
- Issue
    
    할 일, 버그 리포트, 기능 요청 등을 관리하는 기능
    
    → PR과 Issue를 연결하여 작업 관리
- 협업 WorkFlow 

    (Fork & Clone)
    1. Issue 생성
    2. 내 GitHub 계정으로 Fork
    3. Clone으로 코드 가져오기
        >git clone 내_포크_저장소_URL
    4. 원본 저장소 연결
        >git remote add upstream 원본_저장소_URL
    5. 최신 코드 받기
        >git pull upstream main
    6. 브랜치 생성
        >git switch -c feature/add-my-name

    (Collaborator)
    1. Repository collaborator 등록
    2. Branch protection rules 추가
        >Require a pull request before merging
    3. Clone으로 코드 가져오기
        >git clone 원본_저장소_URL
    4. 새로운 브랜치 생성 & 커밋
        >git switch -c feature/mybranch & git commit -m “message”
    5. 원격 브랜치에 push
        >git push origin feature/mybranch
    6. Pull Request 생성
        
        GitHub에서 feature/mybranch → main PR 생성
- git show

    특정 커밋 하나를 집중해서 살펴볼 때 사용하는 명령어
    - 해당 커밋의 변경 내용(diff)
    - 커밋 메시지, 작성자, 날짜 출력
    - 코드 리뷰나 이력 확인 시
    >git show (git show HEAD와 동일)
    
    >git show HEAD (현재 브랜치의 최신 커밋 내용 확인)
    
    >git show "커밋해시"  (특정 커밋 하나를 지정해서 확인)
- git diff

    두 상태 간의 차이점만 비교해서 보여주는 명령어
    - 커밋 전에 변경 사항 점검
    - 병합·충돌 원인 분석
    >git diff (Working Directory ↔ Staging Area)
    
    >git diff --staged (Staging Area ↔ HEAD)
    
    >git diff HEAD~1 HEAD (커밋 ↔ 커밋)
- git blame

    파일의 각 줄이 누가, 언제, 어떤 커밋에서 수정됐는지 추적
    - 버그 원인 추적 시
    - “이 줄은 왜 이렇게 되어 있지?”
    >git blame main.py (main.py의 각 줄을 누가, 언제, 어떤 커밋에서 수정했는지 확인)
- git stash

    현재 작업 중인 변경 사항을 임시로 치워두는 명령어
    - 커밋하지 않고 작업을 잠시 중단
    - 브랜치 이동이나 긴급 수정 시 유용
    >git stash (현재 작업 중인 변경 사항을 임시로 저장)
   
    >git stash list (지금까지 저장된 stash 목록을 확인)
    
    >git stash pop (가장 최근에 저장한 stash를 다시 적용하고 목록에서 제거)
    
    >git stash apply 특정 stash를 제거하지 않고 적용만 하려면
- git reset

    브랜치의 기준을 과거 커밋으로 이동시키는 명령어(히스토리를 바꿈)
    >git reset --soft: 커밋만 취소 (변경 내용 유지)

    >git reset --mixed(기본): 커밋 + 스테이징 취소

    >git reset --hard: 전부 삭제 (협업 브랜치에서 사용시 주의)

    >git reset --soft HEAD~1 HEAD만 한 커밋 이전으로 이동(변경 내용은 유지)

    >git reset --hard HEAD~1 HEAD 이동 + 스테이징 + 파일까지 모두 되돌림
- git revert

    특정 커밋을 되돌리는 새 커밋을 생성(히스토리를 보존)
    - 기존 기록은 그대로 유지
    - “이 커밋을 취소했다”는 기록이 남음
    - 협업 중 안전하게 롤백할 때
    >git revert 커밋해시
- git commit --amend

    마지막 커밋을 고치는 옵션
    - 가장 최근 커밋을 새 커밋으로 교체
    - 커밋 메시지 수정 가능
    - 변경 파일 추가/제거 가능
    >git commit --amend (마지막 커밋을 고침)
    
    >git commit --amend --no-edit (기존 커밋 메시지는 그대로 유지)

##
- HTML (HyperText Markup Language)
    - 웹 페이지의 구조를 정의하는 언어
    - 글의 제목, 문단, 이미지, 링크 같은 웹 페이지의 뼈대를 만드는 역할

- 웹과 HTML
    - 서버가 HTML 문서를 만들어서 브라우저에 전달
    - 브라우저는 HTML을 읽고 화면의 기본 구조를 그림
    - 서버 → HTML 전달 → 브라우저 → 화면 구조 생성
- HTML 기본 구조
    - HTML 문서는 항상 일정한 기본 틀을 가짐
    - 태그(tag)들의 조합으로 구성
        - 태그는 <태그이름> 형태로 작성
        - 각 태그마다 요소의 의미와 역할을 가짐
        - 대부분의 태그는 여는 태그와 닫는 태그로 이루어짐

        >`<h1>제목</h1>` 

        >`<p>문단 내용</p>`

    - 태그 안에 다른 태그가 들어갈 수 있으며, 중첩 구조로 문서의 계층이 만들어짐

```html
<!DOCTYPE html>
<html>
 <head>
 <title>My Page</title>
 </head>
 <body>
 Hello, HTML
 </body>
</html>
```

> `<!DOCTYPE html>`
문서가 HTML5 문서임을 브라우저에 알리는 선언

>`<html>`
HTML 문서의 최상위 태그(모든 HTML 요소는 이 안에 포함)

>`<head>`
화면에 직접 보이지 않는 설정 영역 (제목, 문자 인코딩, CSS, 메타 정보 등)

>`<title>My Page</title>`
브라우저 탭에 표시되는 제목. 검색 결과 제목으로도 사용

>`</head>`
head 영역의 끝

>`<body>`
브라우저 화면에 실제로 보이는 내용 영역. 텍스트, 이미지, 버튼 등이 위치

>`</body> </html>`
각각 body, html 영역의 끝
